Introduction to Ransomware

To clearly define what ransomware is, it’s important to fully understand how it is created and how it affects systems, as well as its behavior and how it infiltrates them.

Ransomware is a type of malware that holds a victim’s personal, confidential, or important information hostage by keeping it encrypted until the owner pays a ransom.

There are several ways this malware can attack a device: through phishing, where the attacker tricks the victim into downloading a malicious file disguised as a PDF or Word document, and through websites that prompt users to scan QR codes or visit pages designed to deliver malware.

A very common entry point is exploiting device vulnerabilities. Attackers first perform reconnaissance to identify weaknesses and system resources, gathering as much information as possible to gain access. Once inside, the malware prepares to avoid detection and establish persistence, making it easier to encrypt files or information and then send a ransom note demanding payment.

How can it be detected?

Now that we’ve learned basic concepts about ransomware, let’s see how it can be detected. According to Fhabte (2023), ransomware attacks can be detected through several common methods:

This is the simplest way to detect a ransomware attack. Signature databases contain indicators that can effectively identify malware on a system.

Examples include file hashes, domain names, and other indicators. Signature-based detection uses a large store of signatures and compares them against files encountered on a system. However, its use has declined because it is not effective against new malware or variants that haven’t yet been added to the signature database.

Behavior-based detection uses algorithms designed to identify malicious activities or unusual actions that do not follow normal patterns.

This approach takes advantage of the fact that ransomware commonly behaves very differently from regular files. For example, during the encryption stage, it opens many files to read their contents and then replaces the data with an encrypted version.

Monitoring file operations and network traffic can reveal threats based on behavior. Ransomware used to perform minimal network operations before encryption, which helped it hide.

Today, many strains exfiltrate or leak data before starting encryption, which creates noticeable network activity and gives attackers leverage for extortion.

How can it be blocked?

Learning how to block and mitigate ransomware is crucial for you and your SME. With SecLab, we’ll review methods to block and mitigate it.

  • Ransomware is malicious code that often disguises itself as email attachments. If an email or a dubious website contains a link, avoid clicking it. Malicious actors commonly infect systems via links that trigger malware downloads.
  • Network segmentation and micro-segmentation for critical systems.
  • Audit privileged group memberships and inherited credentials.
  • There are scanning tools that effectively detect malicious links in emails. These scanners can delete malicious messages before they even reach the device, preventing infections on the device and across the network.
  • Firewalls also act as scanners for file traffic on both sides of a network. They can identify a file’s origin and destination, as well as the method used in transit. Firewalls analyze this data to detect potential threats.
  • Endpoint protection shields individual endpoints from threats. Ransomware can be propagated through malicious applications that infect device endpoints; endpoint protection acts as a shield against these apps.
  • Public Wi-Fi networks are easy for users to access—often without passwords—which makes them convenient but also attractive for cyberattacks and malware propagation.
  • A VPN provides reliable protection by encrypting device data while connected to the internet, creating a “tunnel” through which data travels so that no one can access it except the device’s user.
  • Security software is among the top five ransomware prevention measures. It scans incoming files from the internet. Once malware is detected within a file, the software acts like a wall to prevent the file from accessing the computer.
  • Security software detects potential threats using profiles of known threats and malicious file types. As new threats are discovered, the software updates with this information, becoming more powerful over time.

How to recover a compromised system?

We’ve learned what ransomware is, how to block it, and how to detect it—but what if your device is compromised? That’s why in this section we’ll learn what to do.

  • When a system is infected with ransomware, files are encrypted and a payment is often demanded in exchange for the decryption key. Data can be recovered by following certain steps.
  • First, disconnect the device from the network—both wired and Wi-Fi. This prevents the ransomware from spreading to other devices on the network.
  • Eradicate persistence and patch.
  • There are different types of ransomware, so identify which one it is. This helps determine whether a free decryption tool already exists.

In 2023, the FBI stated that paying a ransom does not guarantee file recovery and finances criminal activity.

The most effective way to recover a system is to restore from a backup made before the attack, so you should maintain backups in case an attack occurs. If there are no backups or decryptors, it may be necessary to format the system and reinstall the OS from scratch, ensuring the new environment is updated and protected.

Types of Ransomware

People often think there are many types of ransomware, but there are two general categories: cryptographic ransomware—the most common—which encrypts data and displays a ransom note; and non-encrypting ransomware, which simply locks the operating system and shows the ransom note. However, these types are divided into sub-categories:

1 · Doxware

Blocks data

Doxware steals personal or sensitive information and threatens to leak it, rather than encrypting it.

2 · Mobile ransomware

Screen locker

As the name suggests, this ransomware targets mobile devices such as phones or tablets. It enters through unauthorized downloads or malicious apps. For these attacks, criminals often choose non-encrypting ransomware so the victim immediately sees the ransom message.

3 · Wiper

Destroyer

This ransomware is less commonly used because its main goal is to steal important data and threaten to destroy it unless a ransom is paid. In extreme cases, the attacker destroys the data even after payment.

4 · Scareware

Pressure on third parties

Scareware tries to trick and frighten the victim into paying or installing something dangerous by sending threatening messages—claiming the user did something illegal (posing as police or government) or insisting they install an “antivirus” because the device is infected—to induce panic so the user feels they have no choice but to pay.

5 · RaaS

Ransomware as a Service

Ransomware as a Service (RaaS) is a cybercrime business model in which ransomware developers sell ransomware code or malware to other hackers—called “affiliates”—who then use the code to launch their own ransomware attacks.

By outsourcing some of their efforts to RaaS providers, aspiring attackers can more quickly and easily enter cybercrime. Even threat actors with limited technical skill can now launch cyberattacks.
